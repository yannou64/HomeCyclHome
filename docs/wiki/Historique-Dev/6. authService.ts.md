# authService.ts

Son rôle : faire les requêtes HTTP liées à l'authentification. C'est tout. Il ne connaît pas React, pas le contexte, pas l'UI. Le service est au bout de la chaîne — il envoie les données et retourne ce que le backend répond.

```
Composant → Hook → Service → API Backend
```

## Gestion des erreurs

Pas de `try/catch` dans le service. Si Axios rencontre une erreur (mauvais identifiants, réseau coupé...), il lève automatiquement une exception qui remonte jusqu'au hook `useLogin`, lequel la traite et expose un message d'erreur au composant.

Si le service attrapait l'erreur lui-même, le hook ne saurait jamais que la requête a échoué.

## Logout

`logout()` appelle le backend pour qu'il supprime le cookie HttpOnly côté serveur. Le frontend seul ne peut pas supprimer un cookie HttpOnly — c'est précisément l'intérêt de ce mécanisme de sécurité.
