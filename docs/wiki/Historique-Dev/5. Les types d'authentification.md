# Les types d'authentification

On distingue deux niveaux de types liés à l'auth :

## `app/providers/authContext/types/auth.types.ts` — types de session

Décrivent ce que l'application **retient** d'une session active. Niveau global, utilisés partout.

- `Role` : les 3 rôles possibles (`client`, `technicien`, `admin`)
- `AuthSession` : le strict minimum d'une session `{ userId, role }` — pas le token (cookie HttpOnly, inaccessible à JS)
- `AuthContextType` : ce que le contexte expose à toute l'application

## `features/auth/types/auth.types.ts` — types de payload

Décrivent ce que l'utilisateur **envoie** dans un formulaire. Spécifiques à la feature auth.

- `LoginPayload` : informations du formulaire de login `{ email, password }`
- `RegisterPayload` : informations du formulaire d'inscription `{ nom, prenom, email, password, telephone }`

## Pourquoi deux fichiers séparés ?

`AuthSession` est une entité de session — elle vit au niveau de l'application.
`LoginPayload` est un DTO d'entrée (Data Transfer Object) — il vit au niveau du formulaire.

Les mélanger créerait une dépendance fragile : si l'entité de session change, ça ne doit pas casser les formulaires.
