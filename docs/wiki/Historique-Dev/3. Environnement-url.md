# Variables d'environnement — VITE_API_URL

Vite injecte les variables d'environnement préfixées `VITE_` dans le code au moment du build. Elles sont accessibles via `import.meta.env.VITE_*`.

## Pourquoi deux valeurs différentes ?

En **développement local**, le frontend (`:5173`) et le backend (`:3000`) tournent sur des ports différents — il faut l'URL complète :

```
# .env  (non commité)
VITE_API_URL=http://localhost:3000/api
```

En **staging et production**, Nginx sert les deux depuis le même domaine et route `/api` vers le backend en interne. Une URL relative suffit :

```
# .env.production  (commitable, pas de secrets)
VITE_API_URL=/api
```

## Comment Vite choisit le bon fichier ?

- `vite dev` → charge `.env`
- `vite build` → charge `.env.production`

Aucun changement de code nécessaire entre les environnements — le bon `VITE_API_URL` est injecté automatiquement au build.
