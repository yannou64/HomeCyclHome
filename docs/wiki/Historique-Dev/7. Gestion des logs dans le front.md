# Gestion des logs dans le frontend

Pour faciliter le débogage, il est utile d'afficher les informations traitées dans la console. Cependant on ne veut pas exposer ces messages aux utilisateurs en production. On utilise une variable fournie par Vite :

```ts
const isDev = import.meta.env.DEV; // true en dev, false après vite build

if (isDev) console.log('[authService] tentative de login :', payload.email);
```

`import.meta.env.DEV` est un booléen automatique — aucune configuration nécessaire.

## Niveaux de log

| Méthode | Usage |
|---|---|
| `console.log` | Flux normal — conditionnel (`isDev`) |
| `console.warn` | Situation anormale mais non bloquante — conditionnel |
| `console.error` | Erreur réelle — toujours visible, même en prod |

## Convention de nommage

Format : `[fichier] action : donnée`

```ts
console.log('[authService] tentative de login :', payload.email);
console.log('[useLogin] session créée, redirection');
console.error('[useLogin] échec login :', err);
```

Le préfixe entre crochets permet de filtrer rapidement dans les DevTools (taper `[auth` affiche uniquement les logs de la couche auth).

## Attention aux données sensibles

Ne jamais logger un payload entier s'il contient un mot de passe :

```ts
// ❌
console.log('[authService] payload :', payload);

// ✅
console.log('[authService] tentative de login :', payload.email);
```
