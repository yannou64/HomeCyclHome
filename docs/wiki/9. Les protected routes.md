# Les routes protégées (ProtectedRoute)

## Le problème

Avec React Router seul, toutes les routes sont accessibles à tout le monde. Un utilisateur non connecté peut taper `/dashboard` dans la barre d'adresse et accéder à la page directement.

`ProtectedRoute` est un composant "garde" — avant de rendre une page, il vérifie la session et redirige si les conditions ne sont pas remplies.

## Principe de fonctionnement

```
Route /dashboard
  └── ProtectedRoute (roles: ['admin'])
        ↓ vérifie session via useAuth()
        ├── pas de session       → redirect /login
        ├── rôle insuffisant     → redirect /unauthorized
        └── session + rôle OK   → render la page
```

## Utilisation dans App.tsx

```tsx
// Route publique — accessible à tous
<Route path="/login" element={<LoginPage />} />

// Route protégée — toute session valide suffit
<Route
    path="/profil"
    element={
        <ProtectedRoute>
            <ProfilPage />
        </ProtectedRoute>
    }
/>

// Route protégée — rôle admin uniquement
<Route
    path="/admin"
    element={
        <ProtectedRoute roles={['admin']}>
            <AdminPage />
        </ProtectedRoute>
    }
/>
```

La prop `roles` est optionnelle :
- Absente → tout utilisateur connecté peut accéder
- Présente → seuls les rôles listés peuvent accéder

## Points techniques

- `<Navigate replace />` — le `replace` évite que le bouton "retour" du navigateur ramène à la page protégée
- `useAuth()` — le composant lit la session depuis le contexte, pas depuis le localStorage directement
- Les rôles disponibles : `'client'` | `'technicien'` | `'admin'`
